Definition Example What it does/doesn’t enable
Level 1: Foundational Different systems can securely exchange data, but the receiving system may not be able to interpret or use the data beyond basic transmission. A hospital sends a discharge summary to a nursing facility via secure email, but the data isn’t structured for easy integration. ✓ Legal, secure data transfer
X integration into workflows
X Risk of information loss or redundancy
Level 2: Structural Data is structured and formatted for receiving systems to display. A lab result is formatted in a way so when it is received by an EHR, it is displayed correctly without manual re-entry. ✓ Fewer manual errors
✓ Faster documentation
Level 3: Semantic Systems can exchange, interpret and meaningfully use data due to common data standards and coding systems that ensure different systems "speak the same language." A cardiologist’s EHR and a primary care physician’s EHR both recognize and categorize a patient’s heart failure diagnosis in the same way, regardless of the software used. ✓ Shared clinical meaning across systems
✓ Improved decision support
✓ Safer referrals and transitions
Level 4: Organizational Data flows seamlessly across institutions, supporting policies, workflows, and governance structures to enable real-time, collaborative care Mayo Clinic Platform Connect enables secure, real-time collaboration across health systems, allowing partners to run AI-driven insights on patients with de-identified clinical data without compromising privacy. ✔ Real-time care coordination
✔ AI and analytics across sites
✔ Reduced duplication and systemwide learning
✔ Highest impact for population health & equity
